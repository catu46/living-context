# Shape: the `knowledge/` bundle + routers + `.context/`

Open this to build a tree. Exact files and copy-paste blocks; reasoning is in [SKILL.md](SKILL.md), the
`.context/` schemas in [context-layer.md](context-layer.md), the type contract in [contract.md](contract.md).

Write everything the skill puts in the user's folders — concept pages, `index.md`, `overview.md`, `log.md`, the
**Rules** in `AGENTS.md` — in the **content/project language**. This reference and the fixed agent-protocol
blocks stay English. Replace every `<placeholder>`. Timestamps are ISO-8601 UTC. Outer fences are `~~~` so nested
``` blocks render.

---

## The model — ONE bundle + per-folder routers + `.context/`

The knowledge is **ONE bundle at `<root>/knowledge/`** (the unit of distribution) with subdirectories **inside**
it that mirror the project layout. On top sit thin `CLAUDE.md`/`AGENTS.md` routers — one per meaningful folder —
that point INTO the bundle. Alongside sits `.context/` (the derived technical layer). `knowledge/` organizes by
**concept kind**, not by mechanically copying the file tree.

```
project-root/
├── CLAUDE.md                     # @AGENTS.md  (the only auto-load bridge)
├── AGENTS.md                     # root router → ./knowledge/index.md
├── knowledge/                    # THE bundle
│   ├── index.md                  #   root map (reserved)
│   ├── overview.md               #   what the project is (the human face of the Project Contract)
│   ├── log.md                    #   ONE append-only history (reserved)
│   ├── knowledge-graph.html      #   ONE graph (generated by scripts/graph.py)
│   ├── systems/                  #   things that RUN (a SQL model, a pipeline, a dashboard feed)
│   ├── processes/                #   how work is done (a recurring analysis, an operating routine)
│   ├── topics/                   #   subject-matter concepts (a metric, a segment, a definition)
│   ├── people/                   #   who/roles (only when it earns a page)
│   └── _archive/                 #   superseded artifact BYTES only — never a concept .md
├── .context/                     # derived, re-creatable (see context-layer.md)
├── sources/meetings/{index.md, inbox/, 2026/}   # when meetings exist
└── <the project's own folders>   # deliverables, analyses, references — routers only, NO per-folder knowledge/
    └── endpoint_sql/             # a work-unit that earns LOCAL docs:
        ├── CLAUDE.md · AGENTS.md · index.md · overview.md · materializa_resultados.sql
```

- **A concept's ID is its path within the bundle** — `knowledge/systems/resultados-xpto.md` → `systems/resultados-xpto`.
- **`source_files:` points at the REAL artifact** (relative path or URL). The concept is a **pointer +
  description**, never a copy.
- **Links** are bundle-relative (leading `/` = bundle root, e.g. `/topics/nrr.md`) or relative (`./sibling.md`).

## 2) `AGENTS.md` router (one per meaningful folder — points INTO the bundle)

Holds ONLY: Rules, a DOWN pointer into this folder's bundle slice, an UP pointer, and "Keep this current". No
definitions, no per-folder `knowledge/`.

~~~markdown
---
type: agent-guide
title: <folder path> — guia do agente
description: <one line: what lives in this folder>
timestamp: 2026-07-01T12:00:00Z
---

# <folder path> — guia do agente

## Rules
- Escopo desta pasta: <o que ela contém e até onde ir>.
- NÃO mexa em: <fontes/arquivos que o agente nunca deve alterar>.

## Knowledge
Conhecimento desta pasta → [`../knowledge/<mirror-path>/index.md`](../knowledge/<mirror-path>/index.md).
Abra só o conceito que precisar.

## If you opened only this folder
Regras globais / mapa raiz → [`../AGENTS.md`](../AGENTS.md) e o mapa do bundle
[`../knowledge/index.md`](../knowledge/index.md). Leia antes de agir se não estiver carregado.

## Keep this current
Depois de mudanças reais aqui, antes de terminar:
1. Atualize o conceito afetado em `knowledge/<mirror-path>/` (ou crie um novo lá).
2. APPEND uma linha em `knowledge/log.md` (nunca reescreva o passado). Restampe o `timestamp` do que tocou.
3. SUPERSEDA — nunca apague — o que foi substituído (template 5).
4. Atualize `.context/` (catalog/graph/manifest/hashes) e regenere o grafo: `python3 <skill-dir>/scripts/graph.py knowledge/`.
5. Criou uma subpasta significativa nova? Adicione o roteador (CLAUDE.md + AGENTS.md) e a fatia do bundle
   `knowledge/<caminho>/index.md`. Se não souber se é significativa ou o que faz, PERGUNTE.
Mude apenas o que a edição tocou.
~~~

**Root router — three changes:** `## Knowledge` points at `./knowledge/index.md`; replace `## If you opened only
this folder` with `## This is the top of the tree` (no `../AGENTS.md` link); and **if the front door is on**, add
the two root-only blocks below.

### Root-only blocks (only when the front door / non-engineer layer is enabled)

~~~markdown
## How to talk to the user
- Reply **in the language the user writes in**, in **plain words** — no jargon.
- Be concise and kind. Say what you're about to do in one sentence before doing it.
- **Confirm before anything destructive** (deleting, overwriting an original, sending). Safe, reversible actions
  (reading, summarizing, creating a new file) — go ahead.
- If something is ambiguous (which version? which source?), **ask** instead of guessing.
- Never touch anything under "Do NOT touch".

## Catch up on changes
When the user opens this folder after a while, or asks "what changed?", reconcile before anything else:
1. `python3 <skill-dir>/scripts/snapshot.py diff .` — added / modified / renamed / deleted since last snapshot.
2. Open each changed file (bounded read) and say in plain words what changed — including changed numbers in a
   re-saved spreadsheet; don't dismiss those as cosmetic.
3. On the user's OK: update the owning concept + `index.md`, APPEND one dated line per change to `log.md`,
   restamp `timestamp`, refresh `.context/`. A rename → repoint the entry, not delete + new.
4. `python3 <skill-dir>/scripts/snapshot.py write .` to refresh the memory.
5. A new meaningful folder appeared with no docs? Scaffold it. Can't tell what it's for? ASK — never invent a
   concept for a folder you don't understand.
~~~

## 8) Local documentation (only in work-units that earn it)

A folder whose technical/operational working needs explaining gets local docs **in place** (not in the bundle):
`index.md` (map/navigation of the folder) + `overview.md` (how the unit works — inputs, outputs, validations) +
the router. The concept page in the bundle links to these via `local_documentation:`. Don't create three
Markdown files in every folder by obligation.

---

## 9) `.context/` stubs

The `.context/` files and their schemas are in [context-layer.md](context-layer.md). At build time create at
least `project-profile.json` (the Contract), `manifest.json` + `hashes.json` (via `snapshot.py`), `catalog.json`
and `graph.json` (from the concepts), and `state.json`. `events.jsonl` and `extracted/` fill in as the engine runs.

---

## Router-only folders (a "mother" holding ONLY subfolders)

A folder with no artifacts of its own — only topic subfolders — is NOT meaningful: it gets **only `CLAUDE.md` +
`AGENTS.md`**, and its `## Knowledge` points DOWN at the subfolders (each self-contained), not at a bundle slice
that wouldn't exist. The root, if router-only, still carries the root-only blocks when the front door is on.
